3) Server-side: make sure the server sends the right headers

For the browser to reliably treat the response as a downloadable file (and to suggest a filename), make sure the server sends:

Content-Type: application/octet-stream
Content-Disposition: attachment; filename="MyGame-Setup.exe"

Examples:

# Express (Node.js)
res.setHeader('Content-Type','application/octet-stream');
res.setHeader('Content-Disposition','attachment; filename="MyGame-Setup.exe"');
res.sendFile(pathToExe);

# nginx location block
add_header Content-Disposition 'attachment; filename="MyGame-Setup.exe"';
Notes: Browsers and security products may warn about `.exe` downloads. Always host .exe files over HTTPS, ensure the server sets CORS if needed, and make sure users trust and expect the download.
